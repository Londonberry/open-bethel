# Explainer site

Static, single-page public site that renders the open-bethel rankings in a way you can read, argue with, and reproduce.

## Files

- **`build.py`** — reads `phase0/games-fhsaa-fl-2026.csv`, computes every ranking + per-game contributions for a curated subset of teams + pairwise connectivity, and writes a single `public/data.json`. Re-run whenever the CSV changes.
- **`public/index.html`** — single page, hash-routed. All views (rankings, team, compare, method, about) share this one HTML shell.
- **`public/styles.css`** — the whole stylesheet. Scientific-almanac aesthetic; warm paper, editorial serif display, tabular-mono data.
- **`public/app.js`** — router + view renderers + utilities. Zero dependencies, zero build step.
- **`public/data.json`** — regenerated by `build.py`. Checked in so GitHub Pages can serve the site without a build step.

## Regenerate

```
python3 site/build.py
```

Takes ~5 minutes on the full FL dataset (most of which is leave-one-out contribution computation for 80 teams).

## Preview locally

```
python3 -m http.server --directory site/public 8000
# → http://localhost:8000
```

## Deploy

`.github/workflows/pages.yml` publishes `site/public/` to GitHub Pages on every push to `main` that changes `site/public/**`. The repo Pages source must be set to "GitHub Actions" in the repository settings (one-time).
